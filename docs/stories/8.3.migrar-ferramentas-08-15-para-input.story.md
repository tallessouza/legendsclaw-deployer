## Story

**As a** operador que usa o deployer em modo interativo ou automatizado,
**I want** que as ferramentas 08 a 15 usem `input()` / `auto_confirm()` no lugar de `read -rp` / `read -rsp`,
**so that** essas ferramentas possam operar em AUTO_MODE lendo valores do config, mantendo o modo interativo identico ao atual.

## Dev Notes

### Pattern de Transformacao

A `lib/auto.sh` (criada na Story 8.1) ja existe em `deployer/lib/auto.sh` e fornece 3 funcoes:

```bash
# input() — substitui read -rp / read -rsp
input "config.key" "Prompt: " variable [--secret] [--required] [--default=VALUE]

# auto_confirm() — substitui read -rp para confirmacoes s/n
auto_confirm "Prompt (s/n): " variable

# auto_load_config() — parseia config file para _AUTO_VALUES
auto_load_config  # usa AUTO_CONFIG env var
```

### Casos Especiais

#### 08-skills.sh — Loops de Validacao com Skills Condicionais

As reads de 08-skills.sh estao dentro de `while true` loops com validacao regex. Em AUTO_MODE, `input()` retorna o valor do config diretamente, entao o loop deve fazer break na primeira iteracao se o valor ja esta valido. Se a validacao falhar em AUTO_MODE, deve retornar exit 1 (config invalido) em vez de re-solicitar.

As reads sao condicionais por skill selecionada (`if is_skill_selected "clickup"`). Em AUTO_MODE, a selecao de skills vem de `skills.selecao` no config (valor "all" ou "1,3,5" etc).

#### 11-seguranca.sh — Menu Interativo de Customizacao

O menu interativo de customizacao (add/remove comandos da blocklist) NAO tem equivalente em AUTO_MODE. Em AUTO_MODE:
- `seguranca.customizar` default "n" → skip inteiro bloco
- Nao ha chaves para add/remove individual — se necessario, customizar blocklist diretamente no script
- Todo o bloco `while true` do menu (opcoes 1-4) deve ser guardado com `if [[ "${AUTO_MODE:-false}" != "true" ]]`

#### 13-evolution.sh — Branching VPS/Local e Password Cascade

- Dois reads identicos de `instance_name` em branches diferentes (VPS linha ~94, local linha ~127) — usar mesma config key
- `url_evolution` so e lido em modo VPS — em AUTO_MODE, config deve fornecer apenas se VPS
- `pg_senha` (unico `read -rsp` nas ferramentas 08-15) — cascade install condicional
- `while read -r` em pipes (linhas ~90, ~123) sao iteracao de display, NAO inputs — nao migrar

#### 15-validacao-final.sh — Teste WhatsApp Opcional

Bloco inteiro e opcional (condicional em Check 9 = OK). Em AUTO_MODE:
- Se `validacao.whatsapp_confirmar: n` → skip todo o bloco com log
- Se `validacao.whatsapp_confirmar: s` → precisa de `validacao.numero_teste` e `validacao.resposta_ok`
- `numero_teste` e sanitizado pos-read (`${numero_teste//[^0-9]/}`) — manter sanitizacao
