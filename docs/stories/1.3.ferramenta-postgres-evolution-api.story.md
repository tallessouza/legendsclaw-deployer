### Dual-Mode: Local vs VPS

**Deteccao:**
```bash
detectar_ambiente() {
    local swarm_state
    swarm_state=$(docker info --format '{{.Swarm.LocalNodeState}}' 2>/dev/null)
    if [[ "$swarm_state" == "active" ]]; then
        echo "vps"
    else
        echo "local"
    fi
}
```

**Deploy wrapper:**
```bash
deploy_stack() {
    local yaml_file="$1"
    local stack_name="$2"
    local ambiente
    ambiente=$(detectar_ambiente)

    if [[ "$ambiente" == "vps" ]]; then
        STACK_NAME="$stack_name"
        stack_editavel  # Portainer API
    else
        docker compose -f "$yaml_file" -p "$stack_name" up -d
    fi
}
```

**Diferencas por modo:**

| Aspecto | Local (compose) | VPS (stack deploy) |
|---------|----------------|-------------------|
| Deploy | `docker compose up -d` | `stack_editavel()` via Portainer API |
| Network | bridge (default) | overlay (externa) |
| HTTPS | Sem (localhost) | Traefik + Let's Encrypt |
| DNS hints | Nao exibe | Exibe tabela A records |
| Wait | `docker compose ps` | `docker service ls` polling |

### YAML Local vs VPS — Diferencas de Geracao

**Modo VPS (Swarm):**
- `version: "3.7"`, networks `external: true` com overlay
- Traefik labels em `deploy.labels` para HTTPS automatico
- `deploy.constraints: [node.role == manager]`
- `deploy.resources.limits` para CPU/memoria

**Modo Local (Compose):**
- Sem Traefik labels (acesso via localhost:porta)
- Network bridge nomeada (nao overlay)
- Ports expostos diretamente (ex: `8080:8080` para Evolution, `5432:5432` para Postgres)
- Sem `deploy` section (compose nao suporta)
- `SERVER_URL=http://localhost:8080` em vez de `https://$url_evolution`

**Estrategia:** O heredoc gera YAML condicional com `if [[ "$ambiente" == "vps" ]]` para as secoes que diferem (Traefik labels, networks, ports, SERVER_URL).

### 8-Step Lifecycle para 03-evolution.sh

1. RESOURCE GATE — `recursos 1 1`
2. LOAD STATE — `dados()`
3. INPUT COLLECTION — dominio Evolution (loop confirmado)
4. RESOLVE DEPENDENCIES — `verificar_container_postgres` → cascata
5. GENERATE YAML — heredoc evolution.yaml (Evolution API + Redis)
6. DEPLOY — `deploy_stack ~/evolution.yaml evolution`
7. VERIFY — `wait_stack evolution_evolution_redis evolution_evolution_api`
8. FINALIZE — salvar dados_evolution + resumo com credenciais
