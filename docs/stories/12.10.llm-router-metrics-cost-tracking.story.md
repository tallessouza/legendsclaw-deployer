## Story

**As a** operador do agente,
**I want** que o LLM Router tenha métricas, cost tracking e skill mapping completo para todas as categorias,
**so that** eu possa monitorar custos e o router selecione o tier correto para cada skill.

## Description

### Contexto Arquitetural

O LLM Router é **nosso componente** (porta 55119), NÃO do OpenClaw. O OpenClaw apenas o vê como custom provider via `models.providers.llm-router` no openclaw.json.

```
OpenClaw Gateway
  → Chama models.providers.llm-router (http://localhost:55119)
  → NOSSO LLM Router decide tier via skill_mapping + keywords
  → Router chama modelo real (DeepSeek, Sonnet, Opus via OpenRouter)
  → Resposta volta pro gateway
```

O `skill_mapping` determina qual tier (budget/standard/quality/premium) usar baseado no contexto. Atualmente, o `generate_skill_mapping()` no 07-llm-router.sh faz pattern matching apenas nas 6 skills interativas. Com a Story 12.5 adicionando ~20 skills novas em 6 categorias, o mapping precisa ser expandido.

### Mudanças

1. **Métricas:** Adicionar `metrics.enabled`, `cost_tracking`, limites por tier
2. **Skill Mapping:** Expandir `generate_skill_mapping()` para cobrir todas as categorias
3. **Tier por Categoria:** Ler o `tier` do SKILL.md (campo no frontmatter) ao invés de pattern match no nome

### Seção Metrics Expandida

```yaml

  enabled: true
  cost_tracking: true
  storage: none
  table: llm_metrics
  batch_size: 10
  flush_interval_ms: 30000
  retention_days: 30
  cost_limits:
    budget: 0.01
    standard: 0.10
    quality: 2.00
    premium: 10.00
  analytics_endpoint: true
```

### Tier por Categoria (derivado do SKILL.md)

| Categoria | Skills | Tier típico |
|-----------|--------|-------------|
| dev | react-best-practices, composition-patterns, etc. | standard |
| infrastructure | clickup-ops, n8n-trigger, etc. | standard |
| memory | context-recovery, todo-tracker, etc. | budget |
| orchestration | planner, task-orchestrator | standard |
| superpowers | brainstorming, writing-plans, TDD, etc. | quality |
| system | model-router, cost-monitor | budget |

### Skill Mapping Expandido (gerado)

```yaml
skill_mapping:
  # Infrastructure (existentes)
  clickup-ops: standard
  n8n-trigger: standard
  supabase-query: standard
  allos-status: budget
  alerts: budget
  budget-control: budget
  # Memory
  context-recovery: budget
  knowledge-graph: standard
  unified-memory: standard
  todo-tracker: budget
  # Orchestration
  planner: standard
  task-orchestrator: standard
  # Superpowers
  brainstorming: quality
  writing-plans: quality
  test-driven-development: quality
  executing-plans: quality
  verification-before-completion: quality
  requesting-code-review: quality
  dispatching-parallel-agents: standard
  systematic-debugging: quality
  # System
  model-router: budget
  cost-monitor: budget
  # Dev
  react-best-practices: standard
  composition-patterns: standard
  web-design-guidelines: standard
  vercel-deploy: standard
```

**Nota:** Este mapping é conceito do NOSSO LLM Router. O OpenClaw não sabe que existe — ele apenas chama `http://localhost:55119/v1/chat/completions` como custom provider. O router decide internamente qual modelo usar baseado no skill + keywords da mensagem.
