## Story

**As a** usuário que quer usar o Legendsclaw na minha máquina local,
**I want** um menu seletor que me direcione para o setup local ao invés do deploy VPS,
**so that** eu consiga preparar meu ambiente local (git, Node.js, Claude Code, Tailscale) de forma automatizada nos 3 SOs principais.

## Description

Criar um menu de seleção de ambiente (Local vs VPS) como primeira tela do deployer, e ferramenta `setup-local.sh` que prepara a máquina local do usuário com as dependências necessárias para os 3 SOs principais (Linux, macOS, Windows via WSL).

O `deployer.sh` atual exibe diretamente o menu de 16 ferramentas VPS. Esta story adiciona uma **tela anterior** que pergunta se o usuário quer:
1. Configurar sua máquina local (novo fluxo)
2. Continuar com o deploy VPS (fluxo existente inalterado)

A ferramenta `setup-local.sh` faz a detecção de SO e instala/verifica as dependências necessárias para o ambiente local:
- **git** (apt/brew/winget)
- **Node.js >= 22** (NodeSource/brew/nvm)
- **Claude Code CLI** (`npm install -g @anthropic-ai/claude-code`)
- **Tailscale** client (curl installer/brew)

### Padrão de env-detect.sh

`detectar_ambiente()` retorna via `echo` (stdout). A nova `detectar_so()` deve seguir o mesmo padrão:
```bash
detectar_so() {
  local kernel
  kernel=$(uname -s)
  case "$kernel" in
    Linux)
      if [[ -n "${WSL_DISTRO_NAME:-}" ]] || grep -qi microsoft /proc/version 2>/dev/null; then
        echo "wsl"
      else
        echo "linux"
      fi
      ;;
    Darwin) echo "macos" ;;
    *)      echo "windows-no-wsl" ;;
  esac
}
```

### Logger Pattern

`lib/logger.sh` é sourced para logging. Usar `log_info`, `log_warn`, `log_error` para registrar em `~/legendsclaw-logs/`. Verificar funções disponíveis antes de usar.

### Nameref Caveat

Bash 4.3+ exigido. Evitar testar namerefs via pipe (quebra). Usar `<<<` redirect.

### hint_tailscale_desktop já existe

`lib/hints.sh` já tem `hint_tailscale_desktop()` que mostra instruções de instalação por SO (Windows/Mac/WSL). A ferramenta setup-local.sh pode reutilizar parcialmente, mas precisa de hint customizado para próximos passos (bridge setup).
