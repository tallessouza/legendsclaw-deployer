## Story

**As a** agente,
**I want** usar o LLM Router para extrair dados e persistir no Memory Manager,
**so that** interpreto respostas inteligentemente e preservo o conhecimento adquirido.

## Acceptance Criteria

1. `process_message` faz extraction call ao LLM Router tier standard
2. Se confianca < 0.7, faz follow-up de clarificacao
3. Se confianca >= 0.7, salva e avanca
4. `export_results` gera User.md, Company.md, TechStack.md
5. Resultados salvos no Memory Manager (`~/.clawd/memory/elicitation/`)
6. Emite evento `elicitation.session.completed` no Event Bus
7. Teste E2E: conversa completa (10-15 msgs) → verificar export

## Dev Notes

### Integracao LLM Router — Contexto Tecnico

Esta story evolui a skill elicitation (Story 4.2) de extracao basica (regex) para extracao inteligente via LLM Router. O LLM Router foi configurado na Story 3.2 e reside em `apps/{agent}/config/llm-router-config.yaml` com 3 tiers.

[Source: docs/prd/epic-4-skills-layer-elicitation.md#story-44]

### LLM Router — Tiers Disponiveis

```yaml
tiers:
  budget:
    models: [deepseek/deepseek-chat]
    provider: openrouter
    max_cost_per_request: $0.01
    use_for: status, health checks, queries simples, formatacao markdown

  standard:
    models: [anthropic/claude-3.5-haiku]
    provider: openrouter
    max_cost_per_request: $0.05
    use_for: operacoes ClickUp, N8N triggers, ANALISES, EXTRACAO DE DADOS

  premium:
    models: [claude-sonnet-4-6]
    provider: anthropic
    max_cost_per_request: $0.20
    use_for: decisoes complexas
```

A extracao de dados (`extractData`) usa tier **standard** (claude-3.5-haiku). A geracao de follow-ups e formatacao markdown usa tier **budget** (deepseek-chat).

[Source: docs/architecture/legendsclaw-architecture.md#llm-router--tiers]

### OpenRouter REST API Pattern

```javascript
const response = await fetch('https://openrouter.ai/api/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${OPENROUTER_API_KEY}`,
    'Content-Type': 'application/json',
    'HTTP-Referer': 'https://legendsclaw.com',  // obrigatorio OpenRouter
    'X-Title': 'Legendsclaw Elicitation'         // opcional
  },
  body: JSON.stringify({
    model: 'anthropic/claude-3.5-haiku',  // tier standard
    messages: [{ role: 'user', content: extractionPrompt }],
    response_format: { type: 'json_object' },
    max_tokens: 500,
    temperature: 0.1  // baixa para extracao precisa
  })
});
```

### Memory Manager — ~/.clawd/memory/

O Memory Manager armazena conhecimento persistente do agente em arquivos markdown em `~/.clawd/memory/`. A skill memory (se habilitada na Story 4.1) ja interage com este diretorio. A story 4.4 adiciona um subdiretorio `elicitation/` com 3 arquivos gerados apos export:

```
~/.clawd/memory/
├── elicitation/
│   ├── User.md        # Dados do fundador
│   ├── Company.md     # Dados da empresa
│   └── TechStack.md   # Stack tecnico
└── ...                # Outros memory files (hooks, etc.)
```
