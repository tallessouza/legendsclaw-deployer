## Story

**As a** operador,
**I want** configurar as skills existentes para minha instancia,
**so that** o agente tenha capacidades operacionais basicas (ClickUp, N8N, Supabase, alertas, status, memory).

## Acceptance Criteria

1. Exibe tabela de skills disponiveis com descricao e dependencias
2. Operador seleciona skills por numero (multi-selecao)
3. Para cada skill selecionada, coleta inputs com hints (URLs exatas de onde obter keys)
4. Atualiza `apps/{agent}/skills/config.js` e `apps/{agent}/skills/index.js`
5. Executa `npm install` no diretorio de skills
6. Testa health check por skill (curl ao servico correspondente)
7. Hints de debug por skill (curl endpoints, onde verificar keys)
8. Salva em `~/dados_vps/dados_skills`
9. Logs salvos em `~/legendsclaw-logs/skills-{timestamp}.log`

### Patterns do Deployer (Lifecycle)

Seguir o lifecycle padrao para ferramenta de setup:
1. LOG INIT — `log_init "skills"` + `step_init N`
2. LOAD STATE — `dados()` + verificar `dados_whitelabel` existe (obrigatorio) + verificar `dados_llm_router` (opcional, WARNING se ausente)
3. CHECK DEPS — verificar `apps/{agent}/skills/` existe
4. EXIBIR TABELA — tabela de skills disponiveis
5. SELECAO — multi-selecao por numero
6. INPUT COLLECTION — loop confirmado por skill (keys + URLs)
7. CONFERINDO AS INFO — resumo com keys mascaradas
8. CREATE SKILL DIRS — criar subdiretorios das skills selecionadas (index.js, SKILL.md)
9. UPDATE CONFIG — atualizar config.js (remover fallbacks SEU_*, adicionar vars faltantes)
10. UPDATE INDEX — atualizar index.js com requires das skills ativas
11. UPDATE .ENV — popular .env com keys
12. NPM INSTALL — instalar dependencias
13. HEALTH CHECK — testar cada skill
14. SAVE STATE + HINTS — dados_skills + hint_skills()

Nao ha resource gate (configuracao de arquivos locais apenas).
Nao ha deploy step (arquivos sao criados/atualizados localmente).
Nao ha source de deploy.sh.

**IMPORTANTE — Local do .env:** O `.env` fica no diretorio de instalacao do OpenClaw (`/opt/openclaw/.env` por default). O path deve ser lido de `~/dados_vps/dados_openclaw` (campo `Install Path`). NAO e na raiz do repo local.

**IMPORTANTE — Backup:** Antes de sobrescrever config.js e index.js, fazer backup com `cp -p`.

**IMPORTANTE — Somente skills selecionadas:** .env e config.js devem conter SOMENTE as variaveis das skills selecionadas pelo operador. Nao incluir todas.

[Source: docs/architecture/legendsclaw-architecture.md#6-arquitetura-do-deployer-script]

### Insights da Story 3.2

- **350 testes baseline** — regressao obrigatoria
- **chmod 600** obrigatorio para arquivos de estado e .env
- **step_ok/step_fail** para feedback visual de cada step
- **Loop confirmado** pattern: while true → read → conferindo_as_info → Y: break / N: restart
- **Keys nunca em JS direto** — sempre via `process.env.VAR_NAME` referenciando .env
