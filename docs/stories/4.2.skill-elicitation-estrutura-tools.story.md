## Story

**As a** desenvolvedor,
**I want** implementar a skill `elicitation` com 4 tools,
**so that** o agente conduza entrevistas estruturadas.

## Acceptance Criteria

1. Cria `apps/{agent}/skills/elicitation/` com SKILL.md, index.js, tools/
2. `start_session`: recebe template_id, cria sessao no Supabase, retorna primeira pergunta
3. `process_message`: processa resposta, extrai dados via LLM, retorna proxima pergunta
4. `get_status`: retorna % completo, campos preenchidos/pendentes
5. `export_results`: exporta JSON estruturado
6. Session state persiste entre mensagens (pausar/retomar)
7. Registra em `skills/index.js`
8. Testes manuais: start → process 3 mensagens → get_status → export

[Source: docs/architecture/legendsclaw-architecture.md#7-arquitetura-da-instância-whitelabel]

### Supabase REST API Pattern

Usar fetch nativo (Node 18+) com headers:
```javascript
const headers = {
  'apikey': process.env.SUPABASE_SERVICE_ROLE_KEY,
  'Authorization': `Bearer ${process.env.SUPABASE_SERVICE_ROLE_KEY}`,
  'Content-Type': 'application/json',
  'Prefer': 'return=representation'
};
const url = `${process.env.SUPABASE_URL}/rest/v1/${table}`;
```

NAO usar `@supabase/supabase-js` — manter lightweight com fetch nativo para reduzir dependencias.

[Source: docs/architecture/legendsclaw-architecture.md#9-integrações-externas]

### Session State — Pausar/Retomar

O session state e persistido inteiramente no Supabase (campo `responses` JSONB + `current_section` + `current_question`). Isso permite:
- Pausar: enviar `__PAUSE__` → status muda para `paused` [Design Decision: @sm — mecanismo escolhido para implementar AC6]
- Retomar: enviar qualquer mensagem quando status e `paused` → status volta para `in_progress`, retoma de onde parou

[Source: docs/prd/epic-4-skills-layer-elicitation.md#story-42 — AC 6]

## Scope

### IN Scope
- Estrutura de diretorios da skill elicitation (index.js, SKILL.md, tools/, lib/)
- 4 tools: start_session, process_message, get_status, export_results
- Supabase REST client lightweight (fetch nativo, sem SDK)
- Session state com pausar/retomar via Supabase
- Extracao basica de dados (regex/pattern matching — SEM LLM nesta story)
- Registro no skills/index.js principal
- Ferramenta deployer 10-elicitation.sh com lifecycle padrao
- Health check de conectividade Supabase

- Criacao das tabelas Supabase (elicitation_templates, sessions, results) — Story 4.3
- Seeds/templates de entrevista — Story 4.3
- Extracao de dados via LLM Router — Story 4.4
- Integracao com Memory Manager — Story 4.4
- Event Bus (elicitation.session.completed) — Story 4.4
- Geracao de User.md, Company.md, TechStack.md — Story 4.4
