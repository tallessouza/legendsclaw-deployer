# 12.4 — Popular Workspace VPS com Todos os Bootstrap Files

```yaml
executor: "@dev"

quality_gate_tools: [code_review, content_validation]
```

## Story

**As a** operador do agente,
**I want** que o workspace do agente contenha **todos os 8 bootstrap files** que o OpenClaw espera,
**so that** o gateway carregue corretamente a persona, tools, heartbeat e memória do agente.

## Description

O OpenClaw carrega **8 bootstrap files** ao iniciar uma sessão de agente (`workspace.ts:308-314`). O deployer atualmente gera **6** (AGENTS.md, SOUL.md, IDENTITY.md, USER.md, BOOTSTRAP.md, MEMORY.md) mas falta:
- **TOOLS.md** — config de tools específicas do agente (SSH hosts, cameras, etc.)
- **HEARTBEAT.md** — padrão de heartbeat session trigger

O OpenClaw auto-cria templates vazios se faltar, mas os templates genéricos não têm o conteúdo customizado pro agente.

### Bootstrap Files do OpenClaw (auditado)

| Arquivo | Função | Deployer gera | Status |
|---------|--------|---------------|--------|
| AGENTS.md | Regras, memory system, always-on skills | Sim | OK |
| SOUL.md | Persona, limites, vibe | Sim | OK |
| IDENTITY.md | Nome, emoji, avatar | Sim | OK |
| USER.md | Perfil do usuário | Sim | OK |
| BOOTSTRAP.md | Primeiro boot (deletado após onboard) | Sim | OK |
| MEMORY.md | Memória longo prazo | Sim | OK |
| **TOOLS.md** | Config de tools (SSH, cameras, filesystem) | **NÃO** | **FALTA** |
| **HEARTBEAT.md** | Heartbeat session trigger pattern | **NÃO** | **FALTA** |

## Dev Notes

### TOOLS.md Template (referência do OpenClaw default)

```markdown
# Tools Configuration

## Environment
- Platform: Linux VPS
- Shell: bash

## SSH Hosts
(Configure SSH hosts the agent can access)

## Filesystem
- Workspace: {workspace_path}
- Logs: {logs_path}

## Restrictions
- No destructive operations without confirmation
- No access to system-critical paths
```

### HEARTBEAT.md Template (referência do OpenClaw default)

```markdown
# Heartbeat

This agent uses periodic heartbeat sessions to maintain context and perform background tasks.

## Trigger Pattern
- Frequency: on-demand
- Mode: passive (only when invoked)

## Background Tasks
- Memory consolidation
- Context recovery check
```

### Referência: `~/.openclaw/workspace/`

O onboard wizard cria workspace default com todos os 8 files. Os templates do onboard são genéricos. O deployer customiza com dados do whitelabel (nome, persona, etc.).
