## Dev Notes

### Arquitetura de Seguranca — 3 Layers

A seguranca do Legendsclaw e organizada em 3 camadas complementares, cada uma operando em nivel diferente:

[Source: docs/architecture/legendsclaw-architecture.md#10-segurança-3-layers]

#### Layer 1: Command Safety (Application Level)

Arquivo `blocklist.yaml` em `apps/{agent-name}/skills/lib/` define comandos bloqueados. Validacao via `command-safety.js` que faz regex matching antes da execucao. Integra com Claude Code Hooks via `bridge.js validate-call` no PreToolUse (Bash) — porem a integracao Bridge e escopo da Story 5.3, nao desta.

[Source: docs/architecture/legendsclaw-architecture.md#layer-1-command-safety-application-level]

#### Layer 2: Docker Sandbox (Container Level)

Imagem Alpine 3.19 com usuario non-root (`sandbox`). Restricoes: `network: none` (sem acesso a rede), `read_only: true` (filesystem somente leitura), `memory: 256m`, `cpu: 0.5`. Stack name: `sandbox`, service: `openclaw-sandbox`. Gerenciado pelo OpenClaw Gateway.

[Source: docs/architecture/legendsclaw-architecture.md#layer-2-docker-sandbox-container-level]

#### Layer 3: Logging/Audit (System Level)

journald com retencao de 6 meses. logrotate com 180 dias, comprimido. Bridge.js registra PostToolUse (Bash) — mas isso e Story 5.3.

[Source: docs/architecture/legendsclaw-architecture.md#layer-3-loggingaudit-system-level]

### Deployer Patterns

O script segue o padrao consolidado de ferramentas do deployer:
- Source libs: ui.sh, logger.sh, common.sh, hints.sh, env-detect.sh
- `log_init` / `log_finish` para logging
- `step_init` / `step_ok` / `step_fail` / `step_skip` para feedback N/M
- `conferindo_as_info` + confirmacao para inputs
- Estado em `~/dados_vps/dados_seguranca`
- `chmod 600` para arquivos com credenciais/config sensivel
- `resumo_final` no ultimo step

[Source: deployer/ferramentas/09-skills.sh, deployer/lib/ui.sh — pattern de ferramentas]

### Hints Pattern

Funcoes de hint seguem o padrao de `deployer/lib/hints.sh`: funcao nomeada `hint_{feature}()`, recebe parametros contextuais, exibe box formatado com dicas praticas. 12 hints ja existem; esta story adiciona 3 novos (blocklist, sandbox, logging).

[Source: deployer/lib/hints.sh — funcoes existentes]
