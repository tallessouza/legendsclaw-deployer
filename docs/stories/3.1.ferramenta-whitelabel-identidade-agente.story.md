## Dev Notes

### Whitelabel Identity — Contexto Tecnico

Esta story implementa a Fase 3 do guide.md: criacao da identidade whitelabel do agente. O objetivo e gerar a estrutura completa `apps/{agent}/` com persona, config e placeholders para skills futuras.

**IMPORTANTE — Separacao Deployer vs AIOS Core:** O deployer roda no VPS e cria a estrutura `apps/{agent}/`. A definicao AIOS do agente (`.aios-core/development/agents/{agent}.md`) e um artefato do framework de desenvolvimento no desktop, e deve ser criada manualmente pelo operador usando Claude Code (`@aios-master *create agent`) ou editando o arquivo diretamente. O deployer NAO gera o agent.md — apenas exibe hint orientando o operador.

[Source: docs/objetivo/guide.md#4-fase-3--customizar-identidade-whitelabel]

[Source: docs/objetivo/guide.md#4.3-customizar-configjs]

### Definicao AIOS do Agente (Manual — Fora do Deployer)

A definicao AIOS do agente e criada no desktop pelo operador, NAO pelo deployer. O arquivo `.aios-core/development/agents/{agent-name}.md` requer o AIOS Core instalado no projeto. O deployer apenas exibe hint orientando o operador a criar este arquivo via `@aios-master *create agent` no Claude Code.

[Source: docs/objetivo/guide.md#4.4-criar-definição-do-agente-aios]

### Patterns do Deployer (Lifecycle)

Seguir o lifecycle padrao para ferramenta de setup:
1. LOG INIT — `log_init "whitelabel"` + `step_init 8`
2. LOAD STATE — `dados()` + verificar `dados_openclaw` existe
3. INPUT COLLECTION — loop confirmado (nome, display name, icone, persona, idioma)
4. CONFERINDO AS INFO — resumo para confirmacao
5. CHECK EXISTING — verificar se `apps/{agent}/` ja existe
6. CREATE STRUCTURE — criar diretorios e arquivos
7. GENERATE CONFIG — config.js com placeholders
8. SAVE STATE + HINTS — append em dados_whitelabel + hint_whitelabel()

Nao ha resource gate (criacao de arquivos locais apenas).
Nao ha deploy step (arquivos sao criados localmente).
Nao ha geracao de agent.md (manual via Claude Code no desktop).

[Source: docs/architecture/legendsclaw-architecture.md#6-arquitetura-do-deployer-script]
