## Story

**As a** operador que roda o deployer na VPS,
**I want** que o 14-gateway-config.sh mergeia configs relevantes E registre o agente no `~/.openclaw/openclaw.json`,
**so that** o gateway OpenClaw realmente use o LLM Router, reconheça o agente, descubra skills e exponha via Tailscale.

## Description

**Bug/Gap atual:** O `14-gateway-config.sh` gera um `aiosbot.json` completo mas o OpenClaw gateway lê `~/.openclaw/openclaw.json` (gerado pelo `pnpm openclaw onboard`). Além disso:
- Nenhum agente está registrado em `agents.list[]`
- Nenhum diretório de skills está em `skills.load.extraDirs`
- O LLM Router não está registrado como custom provider
- `gateway.tailscale.mode` está `"off"` (WSS não funciona)

**Fix:** Após gerar o `aiosbot.json` (mantido como artefato de referência), deep merge no `~/.openclaw/openclaw.json`.

### Agent Registration

O agente deve ser registrado em `agents.list[]` com:

```json
{
  "agents": {
    "list": [
      {
        "id": "{nome_agente}",
        "workspace": "/opt/legendsclaw/deployer/apps/{agent}/workspace",
        "skills": ["memory"]
      }
    ]
  }
}
```

Se `agents.list[]` já existe, fazer merge por `id` (não duplicar, atualizar).

### Skills Discovery

Registrar diretório de skills para auto-discovery:

```json
{
  "skills": {
    "load": {
      "extraDirs": ["/opt/legendsclaw/deployer/apps/{agent}/skills"]
    },
    "entries": {
      "memory": { "enabled": true }
    }
  }
}
```

## Dev Notes

### Deep Merge via Node.js

```javascript
const fs = require('fs');
const openclawPath = `${process.env.HOME}/.openclaw/openclaw.json`;
const aiosbotPath = process.argv[2]; // path to aiosbot.json
const agentId = process.argv[3];     // e.g., "jim"
const agentWorkspace = process.argv[4];
const agentSkillsDir = process.argv[5];

const openclaw = JSON.parse(fs.readFileSync(openclawPath, 'utf8'));
const aiosbot = JSON.parse(fs.readFileSync(aiosbotPath, 'utf8'));

// 1. Merge models.providers
openclaw.models = openclaw.models || {};
openclaw.models.providers = { ...openclaw.models.providers, ...aiosbot.models?.providers };

// 2. Merge agents.defaults (model, models)
openclaw.agents = openclaw.agents || {};
openclaw.agents.defaults = openclaw.agents.defaults || {};
if (aiosbot.agents?.defaults?.model) openclaw.agents.defaults.model = aiosbot.agents.defaults.model;
if (aiosbot.agents?.defaults?.models) openclaw.agents.defaults.models = { ...openclaw.agents.defaults.models, ...aiosbot.agents.defaults.models };

// 3. Register agent in agents.list[]
openclaw.agents.list = openclaw.agents.list || [];
const existingIdx = openclaw.agents.list.findIndex(a => a.id === agentId);
const agentEntry = {
  id: agentId,
  workspace: agentWorkspace,
  skills: aiosbot.agents?.list?.[0]?.skills || undefined
};
if (existingIdx >= 0) {
  openclaw.agents.list[existingIdx] = { ...openclaw.agents.list[existingIdx], ...agentEntry };
} else {
  openclaw.agents.list.push(agentEntry);
}

// 4. Register skills.load.extraDirs
openclaw.skills = openclaw.skills || {};
openclaw.skills.load = openclaw.skills.load || {};
openclaw.skills.load.extraDirs = openclaw.skills.load.extraDirs || [];
if (!openclaw.skills.load.extraDirs.includes(agentSkillsDir)) {
  openclaw.skills.load.extraDirs.push(agentSkillsDir);
}

// 5. Merge skills.entries
if (aiosbot.skills?.entries) {
  openclaw.skills.entries = { ...openclaw.skills.entries, ...aiosbot.skills.entries };
}

// 6. Gateway tailscale mode
openclaw.gateway = openclaw.gateway || {};
openclaw.gateway.tailscale = { ...openclaw.gateway.tailscale, mode: "serve" };

// 7. Gateway auth
if (aiosbot.gateway?.auth) openclaw.gateway.auth = aiosbot.gateway.auth;

// 8. Tools
if (aiosbot.tools) {
  openclaw.tools = openclaw.tools || {};
  if (aiosbot.tools.shell) openclaw.tools.shell = { ...openclaw.tools.shell, ...aiosbot.tools.shell };
  if (aiosbot.tools.filesystem) openclaw.tools.filesystem = { ...openclaw.tools.filesystem, ...aiosbot.tools.filesystem };
}

// 9. Channels
if (aiosbot.channels?.whatsapp) {
  openclaw.channels = openclaw.channels || {};
  openclaw.channels.whatsapp = { ...openclaw.channels.whatsapp, ...aiosbot.channels.whatsapp };
}

// 10. Hooks
if (aiosbot.hooks) openclaw.hooks = { ...openclaw.hooks, ...aiosbot.hooks };

fs.writeFileSync(openclawPath, JSON.stringify(openclaw, null, 2) + '\n');
```

### Observations (CONCERNS — non-blocking)

**OBS-1: Duplicate gateway reload (LOW)**
STEP 12 (line 713) already calls `reload_gateway` when copying configs to workspace. STEP 12c calls it again after merge. On a normal run where both `~/.openclaw/workspace` and `~/.openclaw/openclaw.json` exist, the gateway gets reloaded twice. Not harmful (second reload picks up the merged config), but the first reload is now redundant since the merge hasn't happened yet.

**Recommendation:** Consider removing the `reload_gateway` call from STEP 12 (line 713-715) and relying solely on STEP 12c's reload+healthcheck. This would be a minor cleanup for a future story.

**OBS-2: Test merge code duplication (LOW)**
The Node.js merge logic in `14-gateway-config-merge.bats` (helper `run_merge`) is a copy of the inline code in `14-gateway-config.sh`. If the merge logic changes, both must be updated. Acceptable for now — extracting to a shared `.js` file would be over-engineering at this stage.
