# 12.7 — Script de Validação de Configuração

## Executor Assignment

```yaml
executor: "@dev"
quality_gate: "@qa"

```

## Story

**As a** operador do deployer,
**I want** um script que valide todas as configurações geradas,
**so that** eu possa detectar problemas antes de ativar o agente.

## Description

Criar `deployer/scripts/validate-config.sh` que verifica:
- .env existe com permissões 600
- Placeholders `{{PLACEHOLDER}}` não resolvidos em configs
- JSON syntax de aiosbot.json, node.json, mcp-config.json
- Variáveis obrigatórias presentes
- Secrets vazados (patterns sk-or-v1-, sk-ant- em non-.env)
- Ports disponíveis (18789, 55119)

## Acceptance Criteria

- [ ] AC1: Verifica .env existe com permissões 600
- [ ] AC2: Detecta `{{PLACEHOLDER}}` não resolvidos em configs
- [ ] AC3: Valida JSON syntax de aiosbot.json, node.json, mcp-config.json
- [ ] AC4: Verifica variáveis obrigatórias (GATEWAY_PASSWORD, TAILNET_ID, etc.)
- [ ] AC5: Detecta secrets vazados (patterns sk-or-v1-, sk-ant- em non-.env)
- [ ] AC6: Verifica ports disponíveis (18789, 55119)
- [ ] AC7: Usa libs do deployer (ui.sh, logger.sh) para output consistente

## Scope

### IN
- Novo script `deployer/scripts/validate-config.sh`
- Validação de .env, JSON configs, secrets, ports
- Output usando ui.sh (step_ok/step_fail)

### OUT
- Não corrige problemas automaticamente
- Não valida conectividade (Story 12.8 faz isso)
