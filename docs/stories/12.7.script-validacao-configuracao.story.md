## Story

**As a** operador do deployer,
**I want** um script que valide todas as configurações geradas,
**so that** eu possa detectar problemas antes de ativar o agente.

## Description

Criar `deployer/scripts/validate-config.sh` que verifica:
- .env existe com permissões 600
- Placeholders `{{PLACEHOLDER}}` não resolvidos em configs
- JSON syntax de aiosbot.json, node.json, mcp-config.json
- Variáveis obrigatórias presentes
- Secrets vazados (patterns sk-or-v1-, sk-ant- em non-.env)
- Ports disponíveis (18789, 55119)

## Acceptance Criteria

- [ ] AC1: Verifica .env existe com permissões 600
- [ ] AC2: Detecta `{{PLACEHOLDER}}` não resolvidos em configs
- [ ] AC3: Valida JSON syntax de aiosbot.json, node.json, mcp-config.json
- [ ] AC4: Verifica variáveis obrigatórias no .env:
  - **Sempre obrigatórias:** `AGENT_NAME`, `VPS_IP`, `GATEWAY_HOSTNAME`, `TAILNET_ID`, `GATEWAY_PASSWORD`, `HOOKS_TOKEN`
  - **Pelo menos uma API key:** `OPENROUTER_API_KEY` ou `ANTHROPIC_ADMIN_KEY` ou `OPENAI_API_KEY` ou `GEMINI_API_KEY`
- [ ] AC5: Detecta secrets vazados (patterns `sk-or-v1-`, `sk-ant-`, `sk-proj-`, API keys) em arquivos non-.env dentro de `apps/{agent}/config/`
- [ ] AC6: Verifica ports disponíveis (18789 gateway, 55119 LLM router)
- [ ] AC7: Usa libs do deployer (ui.sh, logger.sh) para output consistente com step_ok/step_fail/step_skip

## Scope

### IN
- Novo script `deployer/scripts/validate-config.sh`
- Criar diretório `deployer/scripts/` (não existe ainda)
- Validação de .env, JSON configs, secrets, ports
- Output usando ui.sh (step_ok/step_fail/step_skip)

### OUT
- Não corrige problemas automaticamente
- Não valida conectividade de rede (Story 12.8 faz isso)
- Não valida estado dos services (systemctl) — Story 12.8

### Padrão do Script

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
DEPLOYER_DIR="${SCRIPT_DIR}/.."
LIB_DIR="${DEPLOYER_DIR}/lib"
source "${LIB_DIR}/ui.sh"
source "${LIB_DIR}/logger.sh"
source "${LIB_DIR}/common.sh"
```

### API de ui.sh

```bash
step_init N           # Inicializa contagem total de steps
step_ok "Mensagem"    # Incrementa OK counter, exibe [OK]
step_fail "Mensagem"  # Incrementa FAIL counter, exibe [FAIL]
step_skip "Mensagem"  # Incrementa SKIP counter, exibe [SKIP]
resumo_final          # Exibe contagem final OK/FAIL/SKIP
```

### Variáveis Obrigatórias do .env (referência 14-gateway-config.sh)

**Sempre obrigatórias:**
- `AGENT_NAME` — Nome do agente
- `VPS_IP` — IP da VPS
- `GATEWAY_HOSTNAME` — Hostname do gateway (ou "localhost")
- `TAILNET_ID` — Identificador da Tailscale tailnet
- `GATEWAY_PASSWORD` — Senha auto-gerada (32 bytes base64)
- `HOOKS_TOKEN` — Token auto-gerado (24 hex bytes)

**Pelo menos uma API key obrigatória:**
- `OPENROUTER_API_KEY`, `ANTHROPIC_ADMIN_KEY`, `OPENAI_API_KEY`, ou `GEMINI_API_KEY`

**Padrão BATS do projeto:**

```bash
#!/usr/bin/env bats

setup() {
  TEST_DIR="$(mktemp -d)"
  LIB_DIR="$(cd "$(dirname "$BATS_TEST_FILENAME")/../../deployer/lib" && pwd)"
  # Source com readonly removido para permitir re-source em testes
  source <(sed 's/^readonly //g' "$LIB_DIR/ui.sh")
}

teardown() {
  rm -rf "$TEST_DIR" 2>/dev/null || true
}

@test "descrição do teste" {
  run comando_a_testar
  [ "$status" -eq 0 ]
  [[ "$output" == *"expected pattern"* ]]
}
```
