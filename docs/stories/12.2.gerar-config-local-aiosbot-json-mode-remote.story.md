# 12.2 — Gerar openclaw-local.json na VPS

## Executor Assignment

```yaml
executor: "@dev"
quality_gate: "@architect"

```

## Story

**As a** deployer que configura o gateway na VPS,
**I want** que a ferramenta 14-gateway-config.sh gere um `openclaw-local.json` com mode:remote,
**so that** o setup-local-openclaw.sh possa copiar esse config para `~/.openclaw/openclaw.json` na máquina local.

## Description

Após gerar o aiosbot.json VPS (existente), o 14-gateway-config.sh gera uma versão local (`apps/{agent}/config/openclaw-local.json`) no formato real do OpenClaw com `gateway.mode: "remote"` e WSS URL do tailnet.

**IMPORTANTE:** Este arquivo é um `openclaw.json` válido (formato nativo OpenClaw), **NÃO** um `aiosbot.json`. O nome é `openclaw-local.json` para distinguir do `openclaw.json` da VPS. O Story 12.1 copia este arquivo para `~/.openclaw/openclaw.json` na máquina local.

## Acceptance Criteria

- [ ] AC1: Após gerar VPS aiosbot.json (existente, inalterado), gera `openclaw-local.json`
- [ ] AC2: Formato nativo OpenClaw (não aiosbot.json): `gateway.mode: "remote"`, `gateway.remote.url: "wss://{hostname}.{tailnet}.ts.net"`, `gateway.remote.password: "{password}"`
- [ ] AC3: Salva como `apps/{agent}/config/openclaw-local.json`
- [ ] AC4: Inclui `agents.defaults` básico (model, workspace)

- [ ] AC6: Step adicional no 14-gateway-config.sh (incrementa step_init)

## Dev Notes

### openclaw-local.json (formato nativo OpenClaw)

```json
{
  "gateway": {
    "mode": "remote",
    "remote": {
      "url": "wss://hostname.tailnet.ts.net",
      "password": "gateway_password"
    }
  },
  "agents": {
    "defaults": {
      "model": { "primary": "openrouter/auto" },
      "workspace": "~/.openclaw/workspace"
    }
  },
  "meta": {
    "lastTouchedVersion": "deployer-12.2",
    "lastTouchedAt": "2026-02-25T00:00:00.000Z",
    "generatedBy": "legendsclaw-deployer"
  }
}
```

**Nota sobre `agents.defaults.workspace`:** No mode:remote, o workspace local é menos crítico — a maioria das operações são executadas no gateway VPS. O workspace default `~/.openclaw/workspace` é suficiente.

### Diferença: openclaw-local.json vs aiosbot.json

| Aspecto | openclaw-local.json | aiosbot.json |
|---------|-------------------|--------------|
| Formato | Nativo OpenClaw | Nosso formato de referência |
| Quem lê | OpenClaw gateway (via ~/.openclaw/) | Nosso agent runtime |
| gateway.mode | `"remote"` | `"local"` |
| Usado por | Story 12.1 (copia para local) | Story 12.0 (merge na VPS) |

### Posição no Script

O novo step deve ser inserido **após Step 7** (geração do `aiosbot.json`) e **antes do Step 8** (geração do `node.json`). As variáveis necessárias já estão disponíveis nesse ponto:
- `ts_hostname`, `ts_tailnet` — extraídos do `dados_tailscale` (Step 3, linha ~75)
- `gateway_password` — gerado no Step 4 (linha ~201)
- `CONFIG_DIR` — definido no Step 5 (linha ~237)
- `openclaw_porta` — extraído do `dados_openclaw` (Step 2, linha ~50)

**Fallback sem Tailscale:** Se `dados_tailscale` não existir, gerar com `"url": "ws://localhost:${openclaw_porta}"` (sem TLS, conexão local direta).
