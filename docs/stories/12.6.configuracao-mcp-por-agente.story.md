# 12.6 — Configuração MCP — Bridge (Claude Code) vs Gateway (Plugins)


## Story

**As a** operador do agente,
**I want** que a configuração MCP esteja correta para cada contexto (Claude Code local vs OpenClaw gateway),
**so that** MCPs funcionem tanto no Claude Code local (via bridge) quanto no gateway (via plugins).

## Description

### Clarificação Arquitetural

Existem **dois contextos diferentes** para MCPs:

| Contexto | Config | Quem lê | MCPs típicos |
|----------|--------|---------|-------------|
| **Claude Code local** | `mcp-config.json` (via bridge/hooks) | Claude Code CLI | brave-search, filesystem, memory |
| **OpenClaw gateway** | `plugins.entries` no `openclaw.json` | OpenClaw gateway | whatsapp, custom plugins |

O `14-gateway-config.sh` atualmente gera `mcp-config.json` — que é correto para o Claude Code local (bridge configura hooks que injetam MCPs). O gateway OpenClaw usa `plugins.entries` no `openclaw.json` (já configurado pelo onboard wizard).

**Esta story é de verificação/validação**, não de implementação nova.

## Acceptance Criteria

- [ ] AC1: `mcp-config.json` gerado corretamente com brave-search, filesystem, memory MCPs
- [ ] AC2: Template placeholders substituídos (BRAVE_API_KEY, SUPABASE_URL)
- [ ] AC3: `mcp-config.json` usado pelo **Claude Code local (bridge)**, NÃO pelo gateway OpenClaw
- [ ] AC4: Gateway OpenClaw usa `plugins.entries` no `openclaw.json` (verificar que whatsapp está configurado)
- [ ] AC5: JSON válido com permissões 600
- [ ] AC6: Documentar claramente nos comments do 14-gateway-config.sh que `mcp-config.json` é para Claude Code, não para gateway

## Scope

### IN

### Formato Esperado do mcp-config.json

O `14-gateway-config.sh` (step 10) deve gerar um JSON neste formato:

```json
{
  "mcpServers": {
    "brave-search": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-brave-search"],
      "env": { "BRAVE_API_KEY": "${BRAVE_API_KEY}" }
    },
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-filesystem", "/opt/legendsclaw"]
    },
    "memory": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-memory"],
      "env": { "SUPABASE_URL": "${SUPABASE_URL}" }
    }
  }
}
```

**Referência:** Verificar o step 10 de `deployer/ferramentas/14-gateway-config.sh` para o formato real atual.
