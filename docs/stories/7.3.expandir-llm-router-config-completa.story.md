## Story

**As a** operador que configura um agente AIOS via deployer,
**I want** que a ferramenta 07-llm-router.sh gere uma configuracao YAML completa equivalente ao stack de referencia (`apps/aiosbot-vps-stack-master/vps/config/llm-router-config.yaml`),
**so that** o agente tenha roteamento inteligente com 4 tiers, keywords com weights, fallback strategy completa e skill mapping granular, maximizando custo-beneficio.

## Acceptance Criteria

1. YAML gerado tem 4 tiers completos (budget, standard, quality, premium) com modelos especificos e `max_cost_per_request` por tier
2. Skill mapping gerado baseado nas skills ativas em `dados_skills` (se nao disponivel, usa defaults do stack de referencia — 14 mappings)
3. Keywords section gerado com 4 categorias e weights (budget:0.3, standard:0.5, quality:0.7, premium:0.9)
4. Fallback strategy completa com `max_retries_per_model` (2), `max_total_retries` (5), `tier_escalation` (true), `anthropic_direct_fallback` (true), `on_error` handlers (rate_limit, timeout, server_error, context_length, invalid_request)
5. Metrics section gerado (enabled: false por default, configuravel via input)
6. Backup do config anterior antes de sobrescrever (`.bak`)
7. Teste de validacao do YAML gerado (parse sem erros via `python3 -c` ou `node -e`)
8. Teste de tier budget com curl (ja existente no script atual, manter funcionando)
