## Dev Notes

### Arquitetura da Validação
[Source: docs/prd/epic-5-whatsapp-security-validation.md#Story 5.4]

A validação final é o último passo do Epic 5. Deve verificar TODOS os componentes instalados nos Epics 1-5 de forma não-destrutiva. Cada check deve ser independente — falha em um não impede verificação dos demais.

### 12 Checks — Fontes de Dados

| Check | Componente | Fonte de Dados | Endpoint/Comando |
|-------|-----------|----------------|------------------|
| 1 | Docker Swarm | Sistema | `docker info --format '{{.Swarm.LocalNodeState}}'` = "active" |
| 2 | Traefik | `dados_vps` | `docker stack ps traefik` — pelo menos 1 running |
| 3 | Portainer | `dados_portainer` | `curl -s https://{PORTAINER_URL}/api/system/status` |
| 4 | OpenClaw Gateway | `dados_openclaw` | `curl -s http://{GATEWAY_URL}/health` → 200 |
| 5 | Tailscale | Sistema | `tailscale status` — exit code 0 |
| 6 | LLM Router | `dados_llm_router` | Verifica arquivo de config e endpoint |
| 7 | Skills | `dados_whitelabel` | Verifica `apps/{AGENT_NAME}/skills/` existe e tem conteúdo |
| 8 | Evolution API | `dados_evolution` | `curl -s http://{EVOLUTION_URL}/manager/instance` |
| 9 | WhatsApp | `dados_evolution` | Instância com `connectionStatus: open` |
| 10 | Security L1 | `dados_whitelabel` | `apps/{AGENT_NAME}/skills/lib/blocklist.yaml` existe |
| 11 | Security L2 | `dados_seguranca` | Definição sandbox Docker presente |
| 12 | Hooks | `dados_bridge` | `HOOKS_CONFIGURED: true` + `bridge.js status` exit 0 |

### Degradação Graceful
[Source: docs/stories/5.3 — Completion Notes]

- Componentes não instalados (dados_* não existe) = `[ SKIP ]`, não `[ FAIL ]`
- VPN offline = check 5 FAIL mas não bloqueia demais checks
- Todos os curl com timeout de 5 segundos (`--connect-timeout 5 --max-time 5`)
- Usar `2>/dev/null || true` em comandos que podem falhar

### Deployer Patterns
[Source: docs/architecture/coding-standards.md + deployer/ferramentas/13-bridge.sh]

```bash
#!/usr/bin/env bash
```

Funções obrigatórias: `log_init`, `step_init`, `step_ok`/`step_fail`/`step_skip`, `resumo_final`
