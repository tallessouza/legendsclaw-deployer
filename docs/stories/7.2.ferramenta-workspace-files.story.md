## Story

**As a** operador que configura um agente AIOS via deployer,
**I want** que a ferramenta 06-workspace.sh gere automaticamente os 6 workspace files do agente (SOUL.md, AGENTS.md, BOOTSTRAP.md, MEMORY.md, IDENTITY.md, USER.md) a partir dos dados coletados pelo whitelabel,
**so that** o agente tenha sua personalidade, instrucoes operacionais e arquivos de workspace prontos para uso imediato, equivalentes ao stack de referencia (`apps/aiosbot-vps-stack-master/`).

## Acceptance Criteria

1. `06-workspace.sh` criado seguindo padrao SetupOrion (source libs, `setup_trap()`, `step_init()`, `log_init()`, `set -euo pipefail`)
2. Carrega dados de `dados_whitelabel` (dependencia: 05-whitelabel.sh) — campos: Agente, Display Name, Icone, Persona, Idioma, Apps Path
3. Gera 6 arquivos no diretorio workspace do agente (`apps/{nome_agente}/workspace/`)
4. `SOUL.md` gerado com persona, estilo e restricoes baseados nos dados coletados — conteudo equivalente ao stack de referencia (42 linhas)
5. `IDENTITY.md` preenchido com nome, display, icone, persona, language a partir de `dados_whitelabel`
6. `USER.md` coleta dados do usuario (nome, nickname, pronouns, timezone, locale) via input interativo ou reutiliza de elicitation se disponivel
7. `AGENTS.md` gerado com lista de skills ativas (referencia `dados_skills` se ja executou 08-skills.sh, senao usa defaults) — conteudo equivalente ao stack de referencia (202 linhas)
8. `MEMORY.md` e `BOOTSTRAP.md` gerados como templates com estrutura correta equivalente ao stack de referencia (16 e 51 linhas)
9. Estado salvo em `dados_workspace` (paths dos 6 arquivos gerados, status, timestamp)
10. `resumo_final()` exibe tabela com arquivos gerados e localizacao

### Padrao SetupOrion para Nova Ferramenta

Seguir exatamente o padrao de `07-whitelabel.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
LIB_DIR="${SCRIPT_DIR}/../lib"

source "${LIB_DIR}/ui.sh"
source "${LIB_DIR}/logger.sh"
source "${LIB_DIR}/common.sh"
source "${LIB_DIR}/hints.sh"
source "${LIB_DIR}/env-detect.sh"

log_init "workspace"
setup_trap

```

- `log_init "nome"` / `log_finish` — inicializa/finaliza logging [Source: deployer/lib/logger.sh#12,45]
