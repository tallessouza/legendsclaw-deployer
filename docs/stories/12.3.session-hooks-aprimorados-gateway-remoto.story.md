# 12.3 — Session Hooks Aprimorados para Gateway Remoto

## Story

**As a** desenvolvedor usando Claude Code com agente remoto,
**I want** que o SessionStart hook verifique Tailscale, Gateway health, OpenClaw config e Bridge status,
**so that** eu saiba imediatamente se toda a stack está funcional ao iniciar a sessão.

## Description

Aprimorar o bloco de hooks JSON no `setup-local-bridge.sh` para que o SessionStart hook faça **4 verificações**:

1. **Tailscale** — VPN mesh conectado (BackendState === "Running")
2. **Gateway** — acessível via HTTP `/health` (bridge check)
3. **OpenClaw config** — `~/.openclaw/openclaw.json` existe e tem `gateway.mode: "remote"` (instalação correta)
4. **Bridge** — serviços descobertos

### Contexto Arquitetural

O hook verifica **duas camadas** distintas:

```
Camada 1: OpenClaw Gateway
  - Config: ~/.openclaw/openclaw.json (gateway.mode: "remote")
  - Conexão: WSS via Tailscale (OpenClaw client gerencia internamente)

Camada 2: Nosso Bridge
  - Health check: HTTP ${GATEWAY_URL}/health
  - Serviços: bridge.js status
```

O gateway health check via HTTP confirma que o gateway VPS está rodando. O OpenClaw client conecta via WSS internamente — o hook não precisa testar WSS diretamente.

## Acceptance Criteria

- [ ] AC1: SessionStart hook verifica: (a) Tailscale conectado, (b) Gateway health via HTTP, (c) OpenClaw config mode:remote, (d) Bridge services
- [ ] AC2: Output estruturado: `[Bridge] Tailscale: OK | Gateway: OK | OpenClaw: remote | Services: N`
- [ ] AC3: Fallback graceful se gateway offline (não bloqueia sessão, mostra warning)
- [ ] AC4: Hooks existentes (PreToolUse, PostToolUse) permanecem inalterados
- [ ] AC5: Se `~/.openclaw/openclaw.json` não existe ou não tem mode:remote, mostra warning

## Scope

### IN
- Modificar bloco `HOOKS_JSON` em `deployer/ferramentas/setup-local-bridge.sh`
- SessionStart hook com 4 checks
- Output formatado e legível

### OUT
- Não modifica PreToolUse ou PostToolUse hooks
- Não instala nada novo
- Não testa WSS diretamente (OpenClaw client gerencia)

### Hook SessionStart Aprimorado

```json
{
  "type": "command",
  "command": "bash -c 'TS=$(tailscale status --json 2>/dev/null | node -pe \"JSON.parse(require(\\\"fs\\\").readFileSync(\\\"/dev/stdin\\\",\\\"utf8\\\")).BackendState\" 2>/dev/null || echo \"offline\"); GW=$(curl -sk --max-time 5 ${GATEWAY_URL}/health 2>/dev/null && echo \"OK\" || echo \"offline\"); OC=$(node -pe \"JSON.parse(require(\\\"fs\\\").readFileSync(require(\\\"os\\\").homedir()+\\\"/.openclaw/openclaw.json\\\",\\\"utf8\\\")).gateway?.mode\" 2>/dev/null || echo \"not configured\"); SVC=$(node .aios-core/infrastructure/services/bridge.js status 2>/dev/null || echo \"no services\"); echo \"[Bridge] Tailscale: $TS | Gateway: $GW | OpenClaw: $OC | $SVC\"'"
}
```

### Diferença: HTTP Health vs WSS Connection

| Check | Protocolo | Quem faz | Story |
|-------|----------|---------|-------|
| Gateway health | HTTP `/health` | Bridge hook (este story) | 12.3 |
| OpenClaw remote | WSS `wss://...` | OpenClaw client (internamente) | 12.1 |
| OpenClaw config | File read | Hook verifica existência/mode | 12.3 |

**Dependência:** Story 12.1 (OpenClaw instalado com `~/.openclaw/openclaw.json`)
