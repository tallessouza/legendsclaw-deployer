## Story

**As a** operador,
**I want** configurar o roteamento de LLMs com tiers de custo,
**so that** o agente use o modelo certo para cada tipo de task, otimizando gastos mantendo qualidade.

## Acceptance Criteria

1. Verifica dependencia: whitelabel existe (`dados_whitelabel`)
2. Coleta: OpenRouter Key, Anthropic Key, DeepSeek Key (opcional), tier padrao
3. Hints com tabela de custos por tier (budget ~$0.01/req, standard ~$0.05/req, premium ~$0.20/req)
4. Atualiza `apps/{agent}/config/llm-router-config.yaml` com providers e API key references
5. Popula `.env` no VPS (`/opt/openclaw/.env`) com as keys e variaveis do LLM Router
6. Testa routing com chamada minima ao tier budget (curl OpenRouter)
7. Exibe resultado do teste + custo estimado
8. Hints de debug (verificar keys, config, curl direto)
9. Salva em `~/dados_vps/dados_llm_router`
10. Logs salvos em `~/legendsclaw-logs/llm-router-{timestamp}.log`

### Arquitetura do LLM Router

| Tier | Provider | Modelo | Custo/req | Uso |
|------|----------|--------|-----------|-----|
| budget | openrouter | deepseek/deepseek-chat | ~$0.01 | status, health, queries simples |
| standard | openrouter | anthropic/claude-3.5-haiku | ~$0.05 | ClickUp ops, N8N triggers |
| premium | anthropic (direto) | claude-sonnet-4-6 | ~$0.20 | decisoes complexas |

**Importante:** O tier premium usa a API Anthropic direta (nao OpenRouter), portanto precisa de key separada.

[Source: docs/architecture/legendsclaw-architecture.md]

### API Keys — Formatos de Validacao

- OpenRouter: prefixo `sk-or-v1-`, minimo 40 chars
- Anthropic: prefixo `sk-ant-`, minimo 50 chars
- DeepSeek: prefixo `sk-`, opcional (atualmente via OpenRouter)

[Source: docs/architecture/legendsclaw-architecture.md#13]

### Patterns do Deployer (Lifecycle)

Seguir o lifecycle padrao para ferramenta de setup:
1. LOG INIT — `log_init "llm-router"` + `step_init 10`
2. LOAD STATE — `dados()` + verificar `dados_whitelabel` existe
3. CHECK DEPS — verificar `llm-router-config.yaml` existe
4. EXIBIR CUSTOS — tabela informativa de tiers
5. INPUT COLLECTION — loop confirmado (keys + tier)
6. CONFERINDO AS INFO — resumo com keys mascaradas
7. UPDATE CONFIG — atualizar llm-router-config.yaml
8. UPDATE .ENV — popular .env com keys
9. TEST ROUTING — curl ao OpenRouter tier budget
10. SAVE STATE + HINTS — dados_llm_router + hint_llm_router()

Nao ha resource gate (configuracao de arquivos locais apenas).
Nao ha deploy step (arquivos sao criados/atualizados localmente).
Nao ha source de deploy.sh.

**IMPORTANTE — Local do .env:** O `.env` fica no diretorio de instalacao do OpenClaw (`/opt/openclaw/.env` por default). O path deve ser lido de `~/dados_vps/dados_openclaw` (campo `Install Path`). NAO e na raiz do repo local.

**IMPORTANTE — Backup:** Antes de sobrescrever `llm-router-config.yaml`, fazer backup com `cp -p` para `llm-router-config.yaml.bak`.

[Source: docs/architecture/legendsclaw-architecture.md#6-arquitetura-do-deployer-script]
