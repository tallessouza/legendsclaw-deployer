# 12.5 — Popular Skills com SKILL.md Discovery + Runtime

## Story

**As a** operador do agente,
**I want** que skills tenham tanto `SKILL.md` (discovery nativo OpenClaw) quanto `index.js` (nosso runtime),
**so that** o OpenClaw descubra as skills automaticamente E nosso agent runtime execute os handlers.

## Description

### Duas Camadas de Skills

O sistema de skills tem **duas camadas distintas** que coexistem:

**Camada 1: OpenClaw Gateway (discovery via SKILL.md)**
- O OpenClaw escaneia diretórios procurando `SKILL.md` em subpastas
- Configurado via `skills.load.extraDirs` no openclaw.json
- Cada skill tem allowlist por agente via `agents[].skills`
- Config por skill via `skills.entries` (enabled, env, apiKey)

**Camada 2: Nosso Agent Runtime (execution via index.js)**
- `index.js` registra skills com handlers e `health()` functions
- `config.js` define configuração de cada skill
- Runtime chama handlers quando skill é invocado
- `skill_mapping` no LLM Router decide tier por skill

### O que precisa mudar no 08-skills.sh

1. **Cada skill ganha `SKILL.md`** — para discovery nativo do OpenClaw
2. **`index.js` permanece** — para nosso runtime de handlers
3. **Story 12.0 registra `skills.load.extraDirs`** no openclaw.json — para OpenClaw encontrar os SKILL.md
4. **Copiar 6 categorias de skills** do template com ambos os formatos

## Dev Notes

### Formato SKILL.md (OpenClaw Discovery)

O OpenClaw procura `SKILL.md` em subdiretórios. O formato mínimo:

```markdown
---
name: memory
description: Long-term memory management for the agent
version: 1.0.0
tier: budget
always_on: false
---

# Memory Skill

Manages agent long-term memory, daily notes, and context recovery.

## Capabilities
- Store and retrieve memories
- Daily note management
- Context recovery after session restart

## Configuration
- Requires: filesystem access to workspace/memory/
```

### Merge do openclaw.json (referência Story 12.0)

O STEP 13b de `14-gateway-config.sh` faz deep merge via Node.js inline. O merge de `skills.entries` já está implementado (linha 836-839):

```javascript
// 5. Merge skills.entries
if (aiosbot.skills?.entries) {
  openclaw.skills.entries = { ...openclaw.skills.entries, ...aiosbot.skills.entries };
}
```

Para AC7, basta garantir que o `aiosbot.json` gerado pelo `08-skills.sh` contenha `skills.entries` com as configs necessárias. O merge no openclaw.json acontece automaticamente quando `14-gateway-config.sh` roda.

**NOTA:** NÃO fazer sync manual para OpenClaw workspace — `skills.load.extraDirs` já aponta direto para `apps/{agent}/skills/`, o OpenClaw descobre os SKILL.md automaticamente.
