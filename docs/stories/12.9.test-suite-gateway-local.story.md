# 12.9 — Test Suite para Gateway e Local

## Executor Assignment

```yaml
executor: "@dev"
quality_gate: "@qa"
quality_gate_tools: [code_review, test_coverage]
```

## Story

**As a** operador do deployer,
**I want** scripts de teste para gateway e local setup,
**so that** eu possa validar que tudo está funcional após instalação.

## Description

Criar dois scripts de smoke test operacionais:
- `deployer/scripts/test-gateway.sh` — testa gateway localhost, Tailscale, endpoint WSS
- `deployer/scripts/test-local.sh` — testa OpenClaw instalado, bridge funcional, hooks configurados, config válido

## Acceptance Criteria

- [ ] AC1: **test-gateway.sh:** Testa gateway localhost (porta 19888), Tailscale endpoint, WSS
- [ ] AC2: **test-local.sh:** Testa OpenClaw instalado, bridge funcional, hooks configurados, `~/.openclaw/openclaw.json` válido
- [ ] AC3: Output com pass/fail por check
- [ ] AC4: Exit code 0 se todos passam, 1 se algum falha

## Scope

### IN
- `deployer/scripts/test-gateway.sh`
- `deployer/scripts/test-local.sh`
- Checks de saúde e conectividade

### OUT
- Não são testes BATS (são scripts de smoke test operacionais)
- Não corrigem problemas

## Tasks / Subtasks

- [ ] **Task 1: Criar `deployer/scripts/test-gateway.sh`**
  - [ ] 1.1 Test `curl localhost:19888/health`
  - [ ] 1.2 Test Tailscale endpoint (se dados_tailscale disponível)
  - [ ] 1.3 Test WSS endpoint (se `gateway.tailscale.mode == "serve"`)
  - [ ] 1.4 Test `agents.list[]` tem agente registrado no openclaw.json
  - [ ] 1.5 Test `models.providers.llm-router` presente no openclaw.json

- [ ] **Task 2: Criar `deployer/scripts/test-local.sh`**
  - [ ] 2.1 Test OpenClaw CLI instalado
  - [ ] 2.2 Test bridge funcional (`node bridge.js status`)
  - [ ] 2.3 Test hooks configurados (`.claude/settings.json` contém bridge.js)
  - [ ] 2.4 Test config válido (`~/.openclaw/openclaw.json` JSON válido)
  - [ ] 2.5 Test `gateway.mode == "remote"` no `~/.openclaw/openclaw.json`
  - [ ] 2.6 Test `gateway.remote.url` começa com `wss://`
