## Story

**As a** operador do deployer,
**I want** scripts de teste para gateway e local setup,
**so that** eu possa validar que tudo está funcional após instalação.

## Description

Criar dois scripts de smoke test operacionais:
- `deployer/scripts/test-gateway.sh` — testa gateway localhost, Tailscale, endpoint WSS
- `deployer/scripts/test-local.sh` — testa OpenClaw instalado, bridge funcional, hooks configurados, config válido

## Acceptance Criteria

- [ ] AC1: **test-gateway.sh:** Testa gateway localhost (porta 19888), Tailscale endpoint, WSS
- [ ] AC2: **test-local.sh:** Testa OpenClaw instalado, bridge funcional, hooks configurados, `~/.openclaw/openclaw.json` válido
- [ ] AC3: Output com pass/fail por check (formato `[✓]`/`[✗]` consistente com validate.sh)
- [ ] AC4: Exit code 0 se todos passam, 1 se algum falha

## Scope

### IN
- `deployer/scripts/test-gateway.sh`
- `deployer/scripts/test-local.sh`
- Checks de saúde e conectividade

### OUT
- Não são testes BATS (são scripts de smoke test operacionais)
- Não corrigem problemas — apenas reportam status

## Dependencies

- **Story 12.0** — merge config no openclaw.json (agents.list, models.providers)
- **Story 12.1** — OpenClaw local mode remote (necessário para test-local.sh)
- **Story 12.2** — openclaw-local.json gerado na VPS (WSS URL)

## Risks

- **LOW:** Gateway pode estar temporariamente down durante restart — script deve ter timeout nos curls (5s)
- **LOW:** Tailscale pode não estar instalado na máquina local — checks de Tailscale devem ser condicionais

### Padrão de Output (seguir validate.sh)

O `validate.sh` (Story 12.8) usa o padrão de steps da lib `ui.sh`:
- `step_init N` — inicializa N steps
- `step_ok "mensagem"` — exibe `[✓] mensagem`
- `step_fail "mensagem"` — exibe `[✗] mensagem`
- `step_skip "mensagem"` — exibe `[-] mensagem` (para checks condicionais)
- Resumo final: `X/N checks passaram`
- Exit code: 0 se todos ok, 1 se algum falhou

### openclaw.json — Campos para Validação

**test-gateway.sh** lê `~/.openclaw/openclaw.json`:
```json
{
  "gateway": { "port": 19888, "tailscale": { "mode": "serve" } },
  "agents": { "list": [{ "id": "nome-agente", ... }] },
  "models": { "providers": { "llm-router": { "baseUrl": "http://localhost:55119" } } }
}
```

**test-local.sh** lê `~/.openclaw/openclaw.json`:
```json
{
  "gateway": {
    "mode": "remote",
    "remote": { "url": "wss://hostname.tailnet.ts.net" }
  }
}
```

Usar `node -e` para parse JSON inline (padrão usado em validate.sh).

### Bridge Status

O bridge tem subcomando `status` confirmado em `.aios-core/infrastructure/services/bridge.js:57-101`:
```bash
node .aios-core/infrastructure/services/bridge.js status 2>/dev/null || echo '[Bridge] Offline'
```
Retorna tabela com status de cada serviço (OK/DEGRADED/FAIL). Exit code 0 se todos OK.

### Notas de Implementação

- Ambos scripts devem ter `#!/usr/bin/env bash` + `set -euo pipefail`
- Source das libs: `source "$(dirname "$0")/../lib/ui.sh"` etc.
- Checks condicionais (Tailscale, WSS) usam `step_skip` quando não aplicáveis
- Timeouts de curl: `--max-time 5` para evitar hang
- `LEGENDSCLAW_TEE_ACTIVE=1` se chamado por outro script (previne tee duplicado)

### Testing

1. Rodar `deployer/scripts/test-gateway.sh` — todos checks devem passar se gateway está up

1. Rodar `deployer/scripts/test-local.sh` — todos checks devem passar se setup local completo
2. Simular falha: renomear `~/.openclaw/openclaw.json` → rodar script → exit code 1

**Não há testes BATS para estes scripts** (são smoke tests operacionais, não unit tests).
