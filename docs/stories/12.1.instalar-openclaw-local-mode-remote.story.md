## Story

**As a** usuário que quer usar o agente conectado ao gateway remoto,
**I want** um script que instale o OpenClaw e configure `~/.openclaw/openclaw.json` com mode:remote na minha máquina local,
**so that** eu consiga conectar ao gateway na VPS via WSS e ter acesso ao agente com skills e workspace.

## Description

Criar `deployer/ferramentas/setup-local-openclaw.sh` que:
1. Instala OpenClaw na máquina local (se não instalado)
2. **Gera `~/.openclaw/openclaw.json` localmente** usando dados do `dados_bridge` (hostname, tailnet, gateway password) — auto-suficiente, sem depender de arquivo da VPS
3. Testa conexão WSS ao gateway VPS

**IMPORTANTE:** O config vai em `~/.openclaw/openclaw.json` (path real do OpenClaw), **NÃO** em `~/.aiosbot/`. O OpenClaw procura config em:
1. `~/.openclaw/openclaw.json` (primário)
2. `~/.openclaw/clawdbot.json` (fallback)
3. Override via `OPENCLAW_HOME` env var

**Integração no install.sh:**
```
Step 6: setup-local.sh (deps)
Step 7: setup-local-bridge.sh (bridge)
Step 8: setup-local-openclaw.sh (OpenClaw remote) ← NOVO
Step 9: setup-local-aios.sh (AIOS + agent registration)
Step 10: setup-local-finish.sh (instruções)
```

## Context

### Sistema Existente

- **install.sh:** One-liner com modo `--local`
- **setup-local.sh:** Instala git, Node.js 22+, Claude Code CLI, Tailscale
- **setup-local-bridge.sh:** Configura Tailscale bridge, hooks Claude Code, service index
- **dados_bridge:** State file com GATEWAY_URL, Tailscale Hostname, Tailnet

### Referência OpenClaw (auditada)

- Config path: `~/.openclaw/openclaw.json`
- Env override: `OPENCLAW_HOME`, `OPENCLAW_STATE_DIR`
- Gateway mode:remote: `gateway.mode: "remote"`, `gateway.remote.url: "wss://..."`, `gateway.remote.password: "..."`

## Dev Notes

### openclaw.json mode:remote (formato real OpenClaw)

```json
{
  "gateway": {
    "mode": "remote",
    "remote": {
      "url": "wss://hostname.tailnet.ts.net",
      "password": "gateway_password"
    }
  },
  "agents": {
    "defaults": {
      "model": { "primary": "openrouter/auto" },
      "workspace": "~/.openclaw/workspace"
    }
  }
}
```

**Nota:** O workspace local pode apontar para `~/.openclaw/workspace` (default) ou para um path customizado. No mode:remote, a maioria das operações são executadas no gateway VPS, então o workspace local é menos crítico.

### State File Format

```
OpenClaw Version: X.Y.Z
Config Path: ~/.openclaw/openclaw.json
Gateway Mode: remote
Gateway URL: wss://hostname.tailnet.ts.net
WSS Test: OK|FAIL|SKIP
Data Configuracao: YYYY-MM-DD HH:MM:SS
```
