## Story

**As a** operador,
**I want** executar um script que prepara a VPS do zero,
**so that** todas as dependencias estejam instaladas antes de qualquer deploy.

## Acceptance Criteria

1. Script executavel via `bash <(curl -sSL ...)` em Ubuntu 22.04
2. Verifica e instala: Docker, jq, apache2-utils, git, python3, Node.js >= 22, pnpm
3. Cada passo exibe feedback `N/15 - [ OK ] - Descricao` ou `[ FAIL ] - Mensagem de erro`
4. Logs salvos em `~/legendsclaw-logs/bootstrap-{timestamp}.log`
5. Verifica se e root, se OS e compativel (soft gate — avisa mas nao bloqueia)
6. Cria estrutura `~/dados_vps/` para estado futuro
7. Se dependencia ja instalada, pula com `[ SKIP ]`

### Deployer Architecture Decisions

- **ADR-005**: Deployer modular sobre monolitico — funcoes em arquivos separados para manutenibilidade
- O bootstrap (setup.sh) e o **primeiro** script do deployer, executavel standalone via curl
- Futuro: deployer.sh tera menu interativo que chama ferramentas individuais
- **DECISAO PO (2026-02-20)**: Na Story 1.1, funcoes utilitarias (feedback, logging, install_if_missing) ficam **inline no setup.sh**. A modularizacao para `deployer/lib/` sera feita na Story 1.2 como pre-requisito, antes de implementar a ferramenta Traefik+Portainer. Isso mantem o setup.sh simples e standalone via curl.

[Source: docs/architecture/legendsclaw-architecture.md#14-decisoes-arquiteturais]
