## Story

**As a** desenvolvedor do agente,
**I want** um system skill que scaffolde novas skills automaticamente,
**so that** eu possa criar novas skills com a estrutura correta (SKILL.md + index.js + tools/).

## Description

Criar um "skill creator" skill na categoria system que gera a estrutura completa de uma nova skill:
- **SKILL.md** — com frontmatter YAML (name, description, version, tier, always_on) + documentação. Formato de discovery nativo do OpenClaw.
- **index.js** — entry point com `handler()` e `health()` (nosso agent runtime)
- **tools/** — diretório para tool implementations (com `.gitkeep`)

O OpenClaw descobre skills escaneando diretórios por `SKILL.md` em subpastas (configurado via `skills.load.extraDirs` no `openclaw.json`). O skill creator gera a estrutura no diretório correto para que o OpenClaw descubra automaticamente — **não existe um registry central `skills/index.js`**.

O skill é invocado via tool `create_skill` registrada no `index.js` do próprio skill-creator, acessível pelo gateway como qualquer outro skill tool.

## Acceptance Criteria

- [ ] AC1: Skill scaffolda nova skill com SKILL.md (frontmatter YAML) + index.js + tools/

- [ ] AC3: Skill creator expõe tool `create_skill` no index.js, invocável pelo gateway como qualquer skill tool

- [ ] AC5: index.js gerado tem `handler()` + `health()` placeholder seguindo o padrão existente (ex.: `memory/index.js`)

### Como o Gateway Descobre Skills

O OpenClaw descobre skills escaneando `SKILL.md` em subpastas dos diretórios configurados em `skills.load.extraDirs` do `openclaw.json`. Quando o skill-creator gera uma nova skill dentro do diretório de skills do agente, o gateway a descobre automaticamente no próximo scan — **não é necessário registro manual**.
