## Story

**As a** operador que usa o deployer em modo interativo ou automatizado,
**I want** que as ferramentas 01 a 07 (e common.sh) usem `input()` / `auto_confirm()` no lugar de `read -rp` / `read -rsp`,
**so that** essas ferramentas possam operar em AUTO_MODE lendo valores do config, mantendo o modo interativo identico ao atual.

## Acceptance Criteria

1. Cada ferramenta (01, 02, 03, 05, 06, 07) adiciona `source "${SCRIPT_DIR}/lib/auto.sh"` e chama `auto_load_config` apos `log_init`
2. Todos os `read -rp` substituidos por `input "ferramenta.key" "prompt" var` com flags adequadas
3. Todos os `read -rsp` substituidos por `input "ferramenta.key" "prompt" var --secret`
4. Todas as confirmacoes (s/n) substituidas por `auto_confirm "prompt" var`
5. `common.sh` — `recursos()` auto-aceita em AUTO_MODE (skip `read`, `escolha="s"`)
6. `common.sh` — `cleanup_on_fail()` skip rollback prompt em AUTO_MODE (apenas loga, `rollback_choice="n"`)
7. Modo interativo (`AUTO_MODE=false`) produz output **identico** ao anterior — zero regressao
8. `deployer/auto-config.example` atualizado com as chaves reais usadas nas ferramentas 01-07
9. BATS existentes continuam passando sem alteracao (`npx bats tests/deployer/`)

## Dev Notes

### Pattern de Transformacao

A `lib/auto.sh` (criada na Story 8.1) ja existe em `deployer/lib/auto.sh` e fornece 3 funcoes:

```bash
# input() — substitui read -rp / read -rsp
input "config.key" "Prompt: " variable [--secret] [--required] [--default=VALUE]

# auto_confirm() — substitui read -rp para confirmacoes s/n
auto_confirm "Prompt (s/n): " variable

# auto_load_config() — parseia config file para _AUTO_VALUES
auto_load_config  # usa AUTO_CONFIG env var
```

**Transformacao mecanica:**
```bash
# ANTES:
read -rp "Dominio do Portainer (ex: painel.exemplo.com): " dominio_portainer

# DEPOIS:
input "base.dominio_portainer" "Dominio do Portainer (ex: painel.exemplo.com): " dominio_portainer --required
```

```bash
# ANTES:
read -rsp "Senha admin do Portainer: " pass_portainer

# DEPOIS:
input "base.pass_portainer" "Senha admin do Portainer: " pass_portainer --secret --required
```

```bash
# ANTES:
read -rp "As informacoes estao corretas? (s/n): " confirmacao

# DEPOIS:
auto_confirm "As informacoes estao corretas? (s/n): " confirmacao
```

### Nota sobre common.sh

`common.sh` NAO deve fazer `source auto.sh` — ela e sourced antes de auto.sh nas ferramentas. Em vez disso, as 2 funcoes (`cleanup_on_fail` e `recursos`) devem verificar `${AUTO_MODE:-false}` diretamente:

```bash
# Em cleanup_on_fail():
if [[ "${AUTO_MODE:-false}" == "true" ]]; then
  rollback_choice="n"
  log "ROLLBACK skipped (AUTO_MODE)" 2>/dev/null || true
else
  read -rp "Deseja executar rollback? (s/n): " rollback_choice </dev/tty || rollback_choice="n"
fi

# Em recursos():
if [[ "${AUTO_MODE:-false}" == "true" ]]; then
  echo "[auto] Recursos insuficientes — aceito automaticamente"
  return 0
else
  read -rp "Continuar mesmo assim? (s/n): " escolha
fi
```

### Nota sobre 07-llm-router.sh

As API keys usam `read -rp` (NAO `read -rsp`) no codigo atual, mas como sao secrets devem ser migradas com `--secret` para que AUTO_MODE as redacte no output.

### Source Order e Inicializacao nas Ferramentas

Padrao existente (nao alterar a ordem, apenas adicionar auto.sh):
```bash
source "${SCRIPT_DIR}/lib/ui.sh"
source "${SCRIPT_DIR}/lib/logger.sh"
source "${SCRIPT_DIR}/lib/common.sh"
source "${SCRIPT_DIR}/lib/auto.sh"     # <-- ADICIONAR
```

Apos `log_init`, adicionar chamada condicional (CRITICO — `set -e` ativo):
```bash
log_init "..."
[[ "${AUTO_MODE:-false}" == "true" ]] && auto_load_config
```
Se chamar `auto_load_config` sem `AUTO_CONFIG` definido, ela retorna exit 1 e o script aborta por `set -e`.
