# 9.1 — install.sh: One-Liner curl|bash para VM limpa

## Story

**As a** operador que quer provisionar uma VM limpa com o Legendsclaw,
**I want** executar um único comando `bash <(curl -sSL <url>/install.sh)` que prepare tudo automaticamente,
**so that** eu não precise clonar manualmente o repo, instalar dependências, ou navegar pela estrutura de arquivos.

## Description

Criar um script `install.sh` na raiz do diretório `deployer/` (futuro repo standalone) que serve como ponto de entrada único para provisionar uma VM limpa. O script é **fino** (~60-80 linhas) — toda a lógica pesada de instalação de dependências já está no `setup.sh`.

O fluxo do script:
1. Verifica pré-requisitos (root, OS, conectividade)
2. Instala `git` se necessário
3. Clona o repo deployer para `/opt/legendsclaw` (ou atualiza se já existe)
4. Executa `setup.sh` automaticamente
5. Exibe instruções para rodar `deployer.sh`

```bash
bash <(curl -sSL https://raw.githubusercontent.com/<org>/legendsclaw-deployer/main/install.sh)
```

### Pattern Visual — SetupOrion N/M
O `setup.sh` já define o pattern de feedback visual que o `install.sh` deve seguir:

```bash
# Formato: N/M - [ STATUS ] - mensagem
# STATUS: OK (verde), FAIL (vermelho), SKIP (amarelo)
feedback() {
  local status="$1"
  local message="$2"
  CURRENT_STEP=$((CURRENT_STEP + 1))
  case "$status" in
    OK)   echo -e "${CURRENT_STEP}/${TOTAL_STEPS} - [ ${GREEN}OK${NC} ] - ${message}" ;;
    FAIL) echo -e "${CURRENT_STEP}/${TOTAL_STEPS} - [ ${RED}FAIL${NC} ] - ${message}" ;;
    SKIP) echo -e "${CURRENT_STEP}/${TOTAL_STEPS} - [ ${YELLOW}SKIP${NC} ] - ${message}" ;;
  esac
}
```
[Source: deployer/setup.sh:59-78]

**IMPORTANTE:** O `install.sh` deve definir sua **própria** função `feedback()` inline (não usar `lib/ui.sh`), pois será executado **antes** do clone — as libs não existem ainda no sistema.

### Cores ANSI — Definição Inline

```bash
readonly RED='\033[0;31m'

readonly CYAN='\033[0;36m'
readonly NC='\033[0m'
readonly BOLD='\033[1m'
```

### Logging Pattern
O `setup.sh` já cria `~/legendsclaw-logs/` e o pattern de log é:
```bash
LOG_DIR="$HOME/legendsclaw-logs"
LOG_FILE="$LOG_DIR/install-${TIMESTAMP}.log"
mkdir -p "$LOG_DIR"
```
[Source: deployer/lib/logger.sh:7-18]

**Nota:** Como o `install.sh` roda antes do clone, o logging deve ser inline (não usar `lib/logger.sh`). Redirecionar stdout+stderr com `exec > >(tee -a "$LOG_FILE") 2>&1`.

### Trap Handler Pattern
```bash
cleanup_on_fail() {
  local exit_code=$?
  if [[ $exit_code -ne 0 ]]; then
    echo -e "${RED}Instalacao falhou no step ${CURRENT_STEP}/${TOTAL_STEPS} (exit code: $exit_code)${NC}" >&2
    echo -e "${RED}Log: ${LOG_FILE}${NC}" >&2
  fi
}
trap 'cleanup_on_fail' EXIT
trap 'echo "Interrompido pelo usuario"; exit 130' INT TERM
```

### Testing

**Framework:** BATS (já usado no projeto em `tests/deployer/`)
[Source: tests/deployer/ — framework existente]

**Pattern de teste BATS:**
```bash
source <(sed 's/^readonly //g' "$LIB_DIR/file.sh")
```

**Cenários de teste positivos:**
1. Execução completa em VM limpa → todos steps OK, exit 0
2. Segunda execução (idempotência) → pull + skip deps, exit 0

**Cenários de teste negativos:**
1. Sem root (`EUID != 0`) → FAIL step 1, exit 1
2. Sem conectividade (github.com inacessível) → FAIL step 3, exit 1
3. `setup.sh` retorna exit code != 0 → FAIL step 6, exit 1
4. `git pull` com conflito (dirty working tree) → FAIL step 5, exit 1
5. Diretório `/opt/legendsclaw` existe mas não é repo git → FAIL step 5, exit 1

**Nota:** Testes unitários das funções inline (feedback, cleanup) podem ser extraídos para teste isolado com BATS. Testes E2E requerem VM/container limpo.
