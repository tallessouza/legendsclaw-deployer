## Story

**As a** operador,
**I want** conectar um número WhatsApp ao meu agente via deployer,
**so that** clientes conversem pelo WhatsApp.

## Acceptance Criteria

1. Gate de recursos: 1 vCPU, 1GB RAM adicional
2. Verifica dependência: OpenClaw Gateway rodando
3. Coleta: domínio Evolution, API Key, número WhatsApp
4. Hints: DNS (registro A), preparação chip eSIM, recargas periódicas
5. Gera `~/evolution.yaml` (Docker Swarm: Evolution API + Redis + Traefik labels)
6. Deploy via `stack_editavel()` (Portainer API)
7. Wait com polling
8. Hints detalhados de pareamento WhatsApp (QR Code step-by-step)
9. Configura webhook Evolution → OpenClaw Gateway
10. Salva em `~/dados_vps/dados_evolution`
11. Hints de debug (Manager URL, curl instâncias, logs Docker)
12. Logs salvos em `~/legendsclaw-logs/evolution-{timestamp}.log`

### Evolution API REST — Endpoints Necessários

```bash
# Base URL: https://{dominio} ou http://localhost:8080
# Auth: Header "apikey: {GLOBAL_API_KEY}"

# Criar instância
POST /instance/create
Body: {
  "instanceName": "legendsclaw",
  "number": "5511999999999",
  "integration": "WHATSAPP-BAILEYS",
  "token": "{instance_token}",
  "webhook": {
    "url": "http://openclaw_gateway:18789/webhook/evolution",
    "byEvents": true,
    "events": ["MESSAGES_UPSERT", "CONNECTION_UPDATE", "QRCODE_UPDATED"]
  }
}

# Conectar (obter QR Code)
GET /instance/connect/{instanceName}
Response: { "base64": "data:image/png;base64,..." }

# Verificar conexão
GET /instance/connectionState/{instanceName}
Response: { "instance": { "state": "open|close|connecting" } }

# Configurar webhook
POST /webhook/set/{instanceName}
Body: {
  "url": "http://openclaw_gateway:18789/webhook/evolution",
  "webhookByEvents": true,
  "events": ["MESSAGES_UPSERT", "CONNECTION_UPDATE"]
}
```

[Source: docs/architecture/legendsclaw-architecture.md#9-integrações-externas — Evolution API]

### Docker Swarm — Webhook Networking

Na rede overlay Docker Swarm (`legendsclaw_net`), os containers se comunicam por nome de serviço. O webhook da Evolution deve apontar para `http://openclaw_gateway:18789/webhook/evolution` (nome do serviço na stack openclaw, não IP).

Para modo local (docker compose), os containers podem não estar na mesma rede. Usar `host.docker.internal:18789` ou criar network compartilhada.

[Source: docs/architecture/legendsclaw-architecture.md#5-arquitetura-de-infraestrutura — Docker Swarm Stacks]
