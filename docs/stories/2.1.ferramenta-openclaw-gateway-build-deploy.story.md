## Story

**I want** instalar o OpenClaw Gateway no VPS via deployer interativo,
**so that** tenha o gateway AI rodando como servico persistente acessivel via Tailscale.

## Acceptance Criteria

1. Gate de recursos: 2 vCPU, 4GB RAM
2. Verifica dependencia: Traefik + Portainer (Story 1.2) ativos
3. Coleta inputs com loop confirmado: dominio gateway, porta (default 18789), repositorio OpenClaw
4. Hints de DNS (registro A para dominio gateway)
5. Clona repo em `/opt/openclaw`
6. Build: `pnpm install` → `pnpm ui:build` → `pnpm build` com feedback por passo
7. Onboard: `pnpm openclaw onboard --install-daemon`
8. Gera systemd unit (`/etc/systemd/system/openclaw.service`)
9. Enable + start servico
10. Health check com retry 5x (`curl http://localhost:18789/health`)
11. Salva em `~/dados_vps/dados_openclaw`
12. Hints de troubleshooting (systemctl, journalctl, ss)
13. Logs salvos em `~/legendsclaw-logs/openclaw-{timestamp}.log`

## Dev Notes

### OpenClaw Gateway — Contexto Tecnico

OpenClaw e o gateway AI central do legendsclaw. Roda como servico systemd (nao como container Docker) na VPS, acessivel via Tailscale mesh na porta 18789.

[Source: docs/prd/epic-2-openclaw-gateway-deploy-tailscale-systemd.md#story-2.1]

### Comandos de Build e Deploy (Referencia)

```bash
# Clonar
git clone https://github.com/openclaw/openclaw.git /opt/openclaw
cd /opt/openclaw

# Build
pnpm install
pnpm ui:build
pnpm build

# Onboard (configura daemon)
pnpm openclaw onboard --install-daemon

# Testar manualmente
pnpm openclaw gateway --port 18789 --verbose

# Verificar
openclaw doctor
openclaw agent --message "Teste" --thinking high
```

[Source: docs/objetivo/guide.md#fase-2-deploy-openclaw-no-vps]

### Systemd Unit (Referencia)

```bash
[Unit]
Description=OpenClaw Gateway
After=network.target tailscaled.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/openclaw
ExecStart=/usr/bin/node openclaw.mjs gateway --port 18789
Restart=always
RestartSec=5
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

[Source: docs/objetivo/guide.md#3.5-systemd-service]

### Diferenca vs Ferramentas Anteriores

- **NAO usa Docker stack deploy** — OpenClaw roda como systemd service nativo
- **NAO usa Portainer API** — systemctl direto
- **Build complexo** — pnpm install + ui:build + build (3 steps sequenciais)
- **Onboard step** — `pnpm openclaw onboard --install-daemon` configura o daemon
- **Pre-requisitos mais pesados** — Node.js >= 22 obrigatorio, pnpm obrigatorio

### 8-Step Lifecycle para 05-openclaw.sh

1. LOG INIT — `log_init "openclaw"` + step_init
2. RESOURCE GATE — `recursos 2 4` (2 vCPU, 4GB RAM)
3. LOAD STATE — `dados()` + verificar `dados_portainer`
4. CHECK DEPENDENCIES — Traefik, Portainer ativos; Node.js >= 22, pnpm, git
5. CHECK EXISTING — OpenClaw ja instalado? → skip
6. INPUT COLLECTION — dominio, porta, repo URL (loop confirmado)
7. INSTALL — clone + build + onboard + systemd unit + enable + start
8. VERIFY — health check retry 5x + dados_openclaw + hints + resumo
