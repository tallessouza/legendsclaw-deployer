## Dev Notes

### Arquitetura Bridge.js
[Source: docs/architecture/legendsclaw-architecture.md#Bridge.js — Auto-Discovery]

```
.aios-core/infrastructure/services/
├── bridge.js                    # Core (escaneia subdiretorios)
└── {agent-name}/
    └── index.js                 # Health check do gateway
        exports: { name, description, health() }
```

Bridge.js e um modulo Node.js que implementa auto-discovery de servicos. Ele escaneia subdiretorios dentro de `.aios-core/infrastructure/services/` procurando arquivos `index.js` que exportem `{ name, description, health() }`.

### Claude Code Hooks
[Source: docs/architecture/legendsclaw-architecture.md#Claude Code Hooks]

| Hook | Trigger | Acao |
|------|---------|------|
| SessionStart | Nova sessao | `bridge.js status` — mostra saude dos servicos |
| PreToolUse (Bash) | Antes de Bash | `bridge.js validate-call` — valida seguranca |
| PostToolUse (Bash) | Apos Bash | `bridge.js log-execution` — audit trail |

### Integracao com Layer 1 (Story 5.2)

O `bridge.js validate-call` integra com a blocklist da Story 5.2:
- Blocklist location: `apps/{agent-name}/skills/lib/blocklist.yaml`
- Se Story 5.2 nao estiver implementada, validate-call deve ser no-op (skip gracefully)

### Deployer Patterns
[Source: docs/stories/5.2 Dev Notes + deployer/ferramentas/09-skills.sh]
