## Story

**As a** usuário que configurou minha máquina local via setup-local (story 11.1),
**I want** uma ferramenta que inicialize o projeto AIOS via `npx aios-core init` e registre meu agente como ativável no AIOS,
**so that** eu consiga ativar meu agente com `@{agent-name}` e usar seus comandos diretamente no Claude Code, com definição completa de persona, commands e dependencies baseada nos dados do deployer.

## Description

Criar a ferramenta `setup-local-aios.sh` que:
1. Verifica pré-requisitos (Node.js ≥ 22, npm)
2. Pergunta nome do projeto e diretório destino
3. Executa `npx aios-core init {nome_projeto}` para inicializar o framework AIOS
4. Verifica que `.aios-core/` foi criado com sucesso
5. Lê dados do whitelabel (se existir) ou coleta interativamente
6. Gera arquivo de definição do agente em `.aios-core/development/agents/{agent-name}.md`
7. Mapeia skills ativas para commands do agente
8. Salva estado e exibe instruções de uso

**Diferença chave vs 12-bridge.sh:** Esta ferramenta não configura conectividade — foca na inicialização do framework AIOS e geração do arquivo de definição do agente seguindo o formato YAML dos agentes existentes.

## Risks

| Risk | Impact | Mitigation |
|------|--------|------------|
| `npx aios-core init` pode mudar de API entre versões | Medium | Verificar exit code e presença de `.aios-core/` após execução |
| `dados_whitelabel` pode não existir (setup local sem VPS) | Medium | Coleta interativa como fallback com defaults razoáveis |
| `dados_skills` pode não existir (nenhuma skill instalada) | Low | Usar set básico de commands (help, status, chat) |
| Formato dos agentes AIOS pode evoluir | Low | Gerar baseado no formato atual, seguindo padrão de dev.md/sm.md |
| `npx` pode estar lento ou falhar por timeout | Low | Retry com mensagem, hint manual como fallback |

### Formato dos Agentes AIOS (Referência para AC7)

O arquivo de definição do agente segue este formato:

```markdown
# {agent-id}

ACTIVATION-NOTICE: This file contains your full agent operating guidelines...

## COMPLETE AGENT DEFINITION FOLLOWS - NO EXTERNAL FILES NEEDED

\```yaml
activation-instructions:
  - STEP 1: Read THIS ENTIRE FILE...
  - ...
agent:
  name: {DisplayName}
  id: {agent-id}
  title: {Title}
  icon: {emoji}
  whenToUse: |
    Use for...
  customization: null
persona_profile:
  archetype: {Archetype}
  communication:
    tone: {tone}
    emoji_frequency: medium
    vocabulary: [...]
    greeting_levels:
      minimal: '{icon} {id} Agent ready'
      named: '{icon} {Name} ({Archetype}) ready...'
    signature_closing: '— {Name}, ...'
persona:
  role: {role description}
  style: {style description}
  identity: {identity description}
  focus: {focus description}
commands:
  - name: help
    visibility: [full, quick, key]
    description: 'Show all available commands'
  - name: status
    visibility: [full, quick, key]
    description: 'Show current status'
  ...
dependencies:
  tools:
    - tool1
    - tool2
\```

## Quick Commands

- `*help` - Show all available commands
- `*status` - Show current status
...
```
