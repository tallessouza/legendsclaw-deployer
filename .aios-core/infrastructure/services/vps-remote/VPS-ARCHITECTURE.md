# VPS Architecture — OpenClaw Remote Reference

## SSH Connection

| Parameter | Value |
|-----------|-------|
| SSH Alias | `openclaw` (configured in `~/.ssh/config`) |
| Default User | `root` |
| Auth | SSH key (`~/.ssh/id_ed25519`) |
| State File | `~/dados_vps/dados_vps_ssh` |

```bash
# Quick test
ssh openclaw echo ok

# Interactive session
ssh openclaw
```

## VPS Directory Layout

```
/opt/openclaw/                  # Main installation
├── bin/                        # Binaries (openclaw, node)
├── skills/                     # Installed skills
│   ├── mcporter/               # MCP Porter skill
│   │   ├── SKILL.md            # Skill manifest
│   │   └── ...                 # Skill files
│   └── <other-skills>/
├── config/                     # System-level config
└── logs/                       # Application logs

~/.openclaw/                    # User-level config
├── openclaw.json               # Main configuration (gateway mode, API keys)
└── skills.json                 # Skills registry
```

## Services

### OpenClaw Gateway (systemd)

```bash
# Service management
systemctl status openclaw
systemctl restart openclaw
systemctl stop openclaw
systemctl start openclaw

# Logs
journalctl -u openclaw -f              # Follow logs
journalctl -u openclaw --no-pager -n 100  # Last 100 lines
```

### Tailscale Serve (HTTPS exposure)

The gateway is exposed via Tailscale Serve on the tailnet:
- Internal: `http://localhost:18789`
- External (tailnet): `https://vps-openclaw.tailca0a3e.ts.net`

```bash
tailscale serve status
```

## Common Operations

### Check Gateway Health

```bash
# From VPS
curl -sf http://localhost:18789/health

# From local (via Tailscale)
curl -sk https://vps-openclaw.tailca0a3e.ts.net/health
```

### Manage Skills

```bash
# List installed skills
ls /opt/openclaw/skills/

# View skill manifest
cat /opt/openclaw/skills/<name>/SKILL.md

# Skills registry
cat ~/.openclaw/skills.json
```

### View Configuration

```bash
cat ~/.openclaw/openclaw.json
```

### Disk Usage

```bash
df -h / /opt/openclaw
du -sh /opt/openclaw/skills/*
```

## VPS Operations Wrapper

Use the `vps-ops.sh` script for convenient access to all operations:

```bash
bash .aios-core/infrastructure/services/vps-remote/vps-ops.sh status
bash .aios-core/infrastructure/services/vps-remote/vps-ops.sh logs 100
bash .aios-core/infrastructure/services/vps-remote/vps-ops.sh health
bash .aios-core/infrastructure/services/vps-remote/vps-ops.sh skills-list
bash .aios-core/infrastructure/services/vps-remote/vps-ops.sh skills-info mcporter
bash .aios-core/infrastructure/services/vps-remote/vps-ops.sh exec "docker ps"
```

All operations are audit-logged to `~/legendsclaw-logs/vps-ops-audit.log`.
